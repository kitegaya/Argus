# Argus Contracts — The 5 Binding Rules

**Everyone must follow these rules exactly.**

---

## Rule 1: Alert Webhook

Alertmanager POSTs to Jenkins. Jenkins extracts `$.alerts[0]` (the first alert only).

The payload received by Jenkins is a single alert object with this shape:
```json
{
  "labels":      { "alertname": "PodCrashLooping", "severity": "critical" },
  "annotations": { "summary": "..." },
  "status":      "firing"
}
```

---

## Rule 2: classification.json

- **Track B writes** this file after the Classify stage.
- **Track C only ADDS** fields — never modifies existing ones.
- `playbook` and `severity` are **immutable** once written.

Required fields (validated by `scripts/validate_contract.py`):

| Field       | Type   | Description                                      |
|-------------|--------|--------------------------------------------------|
| `alertname` | string | Name of the firing alert                         |
| `severity`  | string | `critical` or `warning`                          |
| `playbook`  | string | Ansible playbook name (no `.yml`) or `unknown`   |
| `source`    | string | Always `rules`                                   |

Optional fields Track C may add: `likely_cause`, `ai_enrichment`.

---

## Rule 3: postmortem.json

Track C writes this file after the Document stage by enriching `classification.json`.
Required fields (validated by `scripts/validate_contract.py`):

| Field               | Type   | Description                                  |
|---------------------|--------|----------------------------------------------|
| All Rule 2 fields   | —      | Carried over unchanged from classification   |
| `incident_id`       | string | UUID generated by `store_incident.py`        |
| `timestamp`         | string | ISO-8601 UTC timestamp                       |
| `postmortem_summary`| string | AI-generated or fallback human-readable text |

---

## Rule 4: AWS Names

Track A shares the DynamoDB table name and S3 bucket name with Track C
immediately after `terraform apply`:

```bash
terraform output dynamodb_table_name   # → set as DYNAMODB_TABLE in Jenkins
terraform output s3_bucket_name        # → set as S3_BUCKET in Jenkins
```

---

## Rule 5: Playbook Naming

Every file in `ansible/playbooks/` **must** be registered in `PLAYBOOK_MAP`
inside `scripts/classify_incident.py`. The CI pipeline enforces this automatically.

Current registry:
```python
PLAYBOOK_MAP = {
    'PodCrashLooping': 'restart-deployment',
    'HighMemoryUsage': 'rollback-deployment',
    'HighCPU':         'scale-up',
}
```

---

## Validation

```bash
# Validate individual files against their schemas
python3 scripts/validate_contract.py classification classification.json
python3 scripts/validate_contract.py postmortem    postmortem.json

# CI also validates PLAYBOOK_MAP ↔ ansible/playbooks/ on every push
```
